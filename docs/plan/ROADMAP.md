# ROADMAP

## 목적
- 큰 틀(Phase)과 마일스톤을 고정한다.

## Phase 개요
- P1: 문서/규칙 고정
- P2: 설계 상세화
- P3: 구현
- P4: 공개/운영 최적화

## 마일스톤
- M1: 계획 문서 6종 확정
- M2: 구현 티켓화 + DoD 정의
- M3: Track A 구현/검증
- M4: Track B 구현/검증
- M5: Security Gate PASS 후 공개

## 의존성
- Track B는 Track A 산출물 의존

- 주중: 구현/검증
