# BLOG POST STANDARD

## 목적
- 블로그 글을 일관된 품질/가독성으로 작성하기 위한 표준 구조를 정의한다.

## 기본 톤
- 과장보다 사실 중심
- 후킹은 하되 자극적 과장은 금지
- 줄글 중심 + 섹션 소제목으로 흐름 정리

## 표준 구조
1) 문제 맥락(왜 시작했는가)
2) 목표/원칙(이번 글의 기준)
3) 설계 결정(무엇을 선택했고 왜)
4) 구현 과정(티켓/단계별 핵심 변화)
5) 시행착오/교훈(무엇이 어려웠고 어떻게 풀었는가)
6) 현재 결과(측정 가능한 상태)
7) 다음 단계(남은 과제)

## 글쓰기 규칙
- 소제목은 5~8개 권장
- 각 섹션은 1~3개 문단으로 짧게 유지
- 기술적 사실(파일/경로/정책)은 구체적으로 기록
- 독자에게 유용한 교훈 1~3개 포함

## 게시 전 체크리스트
- [ ] 제목이 과장 없이 후킹되는가
- [ ] 섹션 소제목이 충분한가
- [ ] 구현 사실이 실제 변경과 일치하는가
- [ ] 내부 운영 로그와 공개 글의 경계가 지켜졌는가
- [ ] 다음 단계가 실행 가능하게 적혀 있는가
