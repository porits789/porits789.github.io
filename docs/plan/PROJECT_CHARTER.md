# PROJECT_CHARTER

## 목적
- vibe-coding-study 프로젝트의 존재 이유와 성공 기준을 1페이지로 고정한다.

## 문제 정의
- 바이브코딩 학습은 진행되지만, 학습 결과물이 구조적으로 축적/공유되지 않으면 복습과 재사용이 어렵다.

## 목표(Goal)
- 바이브코딩 공부를 위해 존재한다.
- 공부한 내용을 블로그에 안정적으로 축적/발행한다.

## 비목표(Non-goals)
- 현재 단계에서는 명시적 비목표를 두지 않는다.
- 단, 범위 팽창 방지를 위해 필요 시 즉시 추가한다.

## 범위(In/Out)
- In:
  - 블로그 틀 구성
  - 배포
  - 업로드 체크(기록이 정상 반영되는지 확인)
- Out:
  - TBD (현재 없음)

## 성공 기준 (KPI)
- KPI1: 학습 기록 배포 성공률 90% 이상(주간 기준).
- KPI2: 블로그 글 CRUD(Create/Read/Update/Delete) 핵심 동작 점검 통과(주간 점검 시 실패 0건 목표).
- KPI3: 업로드 후 실제 페이지 반영 확인 완료율 100%(게시 당 확인 체크).
