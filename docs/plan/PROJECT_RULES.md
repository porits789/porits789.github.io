# PROJECT RULES

## 1) 티켓 실행 규칙
- 작업은 T 단위로 진행한다. (T1 → T2 → T3 ...)
- 티켓 범위를 임의로 확장하지 않는다.
- 각 T는 DoD(완료조건) 기준으로 완료 판정한다.

## 2) 완료 처리 규칙 (필수)
각 T 완료 시 아래를 반드시 수행한다.
1. 코드/문서 변경 커밋
2. 원격 저장소 push
3. 진행 문서 업데이트
   - `docs/plan/WORK_STATUS_2026-03-07.md`
   - 관련 계획 문서(필요 시)

## 3) 완료 보고 포맷 (필수)
각 T 완료 보고에는 아래를 포함한다.
1. 완료한 T
2. 변경 파일 목록
3. 실행한 검증 명령 + 결과
4. push 커밋 해시
5. 문서 업데이트 내역
6. 다음 T 계획/예상 소요/블로커

## 4) 사용자 안내 규칙 (추가)
- 사용자 트리거가 없어도 **각 T 완료 시 즉시 사용자에게 안내**한다.
- 안내는 사람 친화 요약으로 작성하며, 내부 툴 노이즈는 제외한다.

## 5) 안전/보안 규칙
- destructive 작업 금지 (명시 승인 전)
- 민감정보/토큰/키 노출 금지
- 보안/공개 관련 이슈는 즉시 ESCALATE 보고
